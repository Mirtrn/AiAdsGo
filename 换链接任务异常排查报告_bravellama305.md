## 三、根本原因

### IPRocket 代理服务商业务异常

错误信息 `"Business abnormality, please contact customer service"` 表明这是 **IPRocket 服务商层面的业务错误**，而非系统技术故障。

### 可能的具体原因

1. **账户配额耗尽** ⭐ 最可能
   - IPRocket 账户流量配额已用完
   - 或达到了请求频率限制（QPS 限制）

2. **账户状态异常**
   - 账户被暂停或限制
   - 可能存在欠费情况

3. **风控触发**
   - 请求频率过高触发风控系统
   - 使用模式被标记为异常

4. **服务商临时故障**
   - IPRocket 服务端临时性故障
   - 特定地区节点服务中断

## 四、解决方案

### 🔴 立即行动（用户侧）

#### 1. 检查 IPRocket 账户状态

请按以下步骤操作：

```
✓ 登录 IPRocket 控制台 (https://iprocket.io)
✓ 检查账户余额和流量配额
✓ 查看是否有风控提示或账户限制通知
✓ 检查账单和付款状态
✓ 如有问题，联系 IPRocket 客服确认账户状态
```

#### 2. 临时解决方案

如果 IPRocket 账户确实有问题，可以考虑：

**方案A：充值/升级**
- 充值 IPRocket 账户
- 或升级到更高配额的套餐

**方案B：降低请求频率**
- 暂时禁用部分任务
- 增加任务执行间隔（从设置中调整）

**方案C：更换代理服务商**
- 考虑使用其他代理服务商：
  - Oxylabs
  - Bright Data
  - Smartproxy
  - 其他支持的服务商

#### 3. 重新启用任务

在解决 IPRocket 账户问题后：

```
1. 进入"换链接"页面
2. 找到状态为 "error" 的任务
3. 点击任务进入详情页
4. 点击"重新启用"按钮
5. 系统会重置连续失败计数器，任务恢复正常执行
```

需要重新启用的任务：

## 七、技术细节说明

### 系统的错误处理机制

系统已经正确识别并处理了 IPRocket 业务错误：

1. **错误识别**：系统能够区分 IPRocket 业务错误和其他技术错误
2. **智能重试**：对于业务错误，系统不会进行无效重试
3. **自动保护**：连续失败3次后自动标记为 error 状态，避免持续消耗资源
4. **错误分类**：将错误分类为 `link_resolution` 类型，便于后续分析
